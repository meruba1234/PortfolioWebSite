BIツール × 仕入先出荷目安 突合レポート

主要仕入先からの「出荷目安数」エクセルと、社内BI (Dr.Sum) の前年同月売上を突き合わせ、課別ケース売上ベースで「どれだけ出荷可能か / 不足か」を判定するレポートを自動生成。

業種
包装資材卸 (仕入先: 主要メーカー)
対象部門
営業部 (上長・課別)
実装期間
約3週間
本番稼働
2026年5月〜
技術スタック
Python, Playwright (Dr.Sum 自動DL), openpyxl, SQL Server, Prefect 3.x

背景・課題

毎月初、仕入先メーカーから「出荷目安数」のエクセルが届きます。 これを社内の前年同月売上 (Dr.Sum BIツール) と突き合わせて、各課がどれだけ出荷可能か/不足かを確認するレポートが必要でした。

従来の課題:

  • Dr.Sum から3社分 (グループ会社含む) の前年同月売上を手動ダウンロード
  • 福助エクセルと商品コード突合 (グループ会社で商品コード体系が異なる)
  • 商品マスタから換算値3 (ケース入数) を取得して、本数換算からケース換算に変換
  • 上記すべてを Excel で手作業 → 月初の貴重な営業時間を消費

実装した解決策

1. BIツール自動ダウンロード

  • Playwright で Dr.Sum にログインし、レポート画面で条件を設定して Excel ダウンロード
  • 日付入力フォーマット (YYYYMMDD 区切りなし) の罠を検証 → 仕様書化
  • 3社分 (本社・グループA・グループB) を一気に取得

2. 商品コード変換 (互換コード機構)

  • 仕入先側の商品コードと自社側の商品コードを変換する「互換コード」シートを別途管理
  • 変換テーブルを外部ファイル (xlsx) で持ち、業務側がメンテできる設計
  • グループ会社間の商品コード体系の違いも吸収

3. ケース換算

  • DB の M_商品 から換算値3 (ケース入数) を取得
  • 本数ベースの売上をケース換算に変換して比較

4. 出力ファイルの工夫

  • ファイル名に「N月分」「作成日時」を埋め込み (上書き事故防止)
  • archive フォルダに処理済みエクセルを移動 (重複検出は内容ハッシュ)
  • 妥当性チェック結果を ChatLuck に WARNING/ERROR 通知

5. アーキテクチャ

  • Flow 層 (Prefect 統合 + subprocess 起動) と Script 層 (業務ロジック) を別リポジトリに分離
  • Script 層は他ジョブ (大三西澤商店レポート等) でも再利用できる設計

効果

項目BeforeAfter
BIツールから売上 DL 3社 × 手動 ≒ 30分 自動 (約2分)
商品コード突合 手作業で照合 互換コード自動変換
ケース換算 手計算で電卓 DB 自動取得
合計レポート作成時間 約2時間/月 約5分/月 (DL→確認のみ)
計算ミスリスク 手計算でミス発生しうる 自動化でゼロ

得られた学び

  • 業務側がメンテできる箇所を残す設計 — 互換コードシートを外部ファイル化することで、商品体系の変更を業務側で対応可能に
  • Flow 層と Script 層の責務分離 — Prefect 統合部分と業務ロジックを分けることで、他ジョブでの再利用と単体テスト容易性を確保
  • BIツールの入力フォーマット罠を仕様書化 — YYYYMMDD 区切りなし、というハマりポイントは将来の自分のために必ず文書化

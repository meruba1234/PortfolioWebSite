売上CSV取込システム (VB.NET → Python 移植)

5,489 行に及ぶ販売管理システムの VB.NET 業務ロジックを解読し、Python に移植して Prefect 上で運用。10分間隔ポーリングで自動取り込み。

業種
卸売業 (社内基幹システム)
対象部門
営業事務・受注処理担当
実装期間
VB.NET 解析 + Python 移植 = 数ヶ月 (継続中)
本番稼働
2026年5月 (235サーバー移行 Phase 1-2 完了)
技術スタック
Python (csv_to_db_v3.py, 5,489行), VB.NET 解析 (frmMain.vb), SQL Server (pyodbc), Prefect 3.x

背景・課題

受注 CSV を販売管理 DB に登録する処理は、長年「振分け表エクセル (VBA)」と「frmMain.vb」の2つのシステムに依存していました。

  • VBA: 5,000行超のマクロで CSV を整形
  • frmMain.vb: 整形後のデータを DB に登録 (金額計算ロジック含む)

このまま運用を続けると:

  • VB.NET / VBA を読める人が社内にいなくなる → ベンダー依存
  • 業務ルール変更の度に高額な改修費用
  • 古いOSのサポート切れリスク

これを Python に移植して、社内で自由に改修できる体制を構築する必要がありました。

実装した解決策

1. VB.NET ソースコードの解読 (リバースエンジニアリング)

  • frmMain.vb (約5,000行) を逐次解読
  • 商品マスタ・得意先マスタ・単価マスタ・EOS 換算・税率の参照ロジックを整理
  • 金額計算式 (単価 × 数量、内税・外税、軽減税率対応など) を抽出
  • 業務ルール (得意先別の特殊処理、商品別のEOS振り分け) を仕様書化

2. Python 互換実装 (csv_to_db_v3.py)

  • 5,489 行の Python コードで VB.NET ロジックを完全互換実装
  • ReadonlyDb + Repository パターンで読み取りと書き込みを分離
  • 得意先 / 商品 / 単価 / EOS換算 / 税率 を Repository クラスで管理
  • frmMain.vb の金額計算ロジックを Python に移植 (税抜・税込・軽減税率・割戻すべて対応)

3. Prefect への組み込み

  • 10分間隔で処理前フォルダをポーリング
  • 多重起動防止 (concurrency_limit=1)
  • 共有フォルダ (UNC) からの読み取りを batch-files/common/shared_fs.py で透過化

4. 段階移行 (89 → 235 サーバー)

  • Phase 1: ローカル Windows PC で動作確認
  • Phase 2: 235 Linux サーバーで試験運用 (195行 → 42伝票の試験データで3回連続 Completed 確認)
  • Phase 3: 旧コードからの完全移行 (継続中)

5. 設定の Variable / Secret 化

  • DB 接続情報を Prefect Secret Block で管理
  • サーバー / DB 名を Prefect Variable で管理
  • 共有 .env からの脱却計画

効果

項目BeforeAfter
業務ロジック改修 ベンダー依頼 (数十万円〜) 社内で完結
処理基盤 Windows PC 上の VB.NET / VBA Linux Docker 上の Python (Prefect)
保守性 古い OS とコードに依存 モダンスタック (Python 3.x)
ベンダー依存度 改修の度に高額見積もり 完全脱却
処理速度 遅い (VB.NET の同期処理) 高速化済み (pyodbc + 一括処理)

得られた学び

  • VB.NET / VBA の解読力は希少価値 — レガシーシステムを読める人が減っている今、解読してモダン言語に移植できる人材は社内外で重宝される
  • 段階移行の厳格な検証 — 195行 → 42伝票のサンプルで3回連続 Completed を確認してから本番投入。「動いた、終わり」ではなく「同じ結果が出ることを証明」
  • Repository パターンでドメインを整理 — 商品 / 得意先 / 単価 / EOS / 税率を Repository クラスで分離することで、他ジョブからも再利用可能に
  • 業務ロジックの仕様書化 — 5,489行の Python コードを書くだけでなく、業務ルールを別途文書化することで、将来の改修者を支援

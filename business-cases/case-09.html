FAX 履歴ビュアー (Web ツール)

複合機の FAX 履歴を社内 Web から検索・閲覧できるツール。複合機側は直近データしか保持しないため、定期 DL して日付別に蓄積。トラブル時の追跡が一瞬で完了する。

業種
卸売業 (社内インフラ)
対象部門
全社 (営業・営業事務・情シス)
実装期間
初期版 + 89→235 移植 = 数ヶ月
本番稼働
2026年4月 (235 サーバー Cutover完了 2026年5月)
技術スタック
Python (Flask), Playwright (複合機Web管理), Prefect 3.x, Docker, nginx, タスクスケジューラ (旧)

背景・課題

FAX 送信履歴は複合機の Web 管理画面でしか確認できず、直近データしか保持されません。

トラブル発生時 (取引先から「FAX 届いてない」と連絡があった等) には、過去の送信状況をすぐ確認したいが:

  • 複合機にログインしないと見られない
  • 直近データのみ → 過去のトラブル追跡が困難
  • 営業がそれぞれ覚えていない

「過去 FAX を一発で検索したい」というニーズが恒常的にありました。

実装した解決策

1. 履歴ジョブ (Prefect 側)

  • タスクスケジューラから 1 分おきに pythonw.exe で起動 (月〜土 7:00〜24:00)
  • Playwright で複合機 Web 管理画面にログインし、ジョブ履歴 CSV をダウンロード
  • 日付別・カテゴリ別に CSV として共有フォルダに蓄積
  • 前回実行中の場合は IgnoreNew でスキップ

2. Web ビュアー (Flask)

  • 蓄積した CSV をブラウザから検索・閲覧
  • 日付指定・送信先指定・キーワード検索
  • nginx リバースプロキシで社内 URL から提供

3. FAX アドレス帳同期 (関連ジョブ)

  • マスター複合機の宛先表 CSV を定期取得
  • 合体版アドレス帳 abook_merged.csv を生成して共有フォルダに配布
  • 各 PC のプリンタアドレス帳を統一

4. 89 → 235 サーバー Cutover

  • 当初は 89 ローカル PC で運用 (タスクスケジューラ + viewer 一体型)
  • 2026年5月、Ubuntu Linux サーバー (235) に Cutover
  • 責務分離: Prefect Flow が CSV 取得・マージ、Flask Server が UI 提供
  • 旧 URL 8080 リダイレクトは 2026/6/1 で廃止

5. UI 改善 (継続中)

  • アドレス帳変更ボタンを直リンク化 (将来計画: ワンクリックで宛先表編集画面まで)

効果

項目BeforeAfter
過去 FAX の確認 複合機ログイン必要・直近データのみ Web ブラウザで全期間検索
FAX トラブル時の確認時間 10〜30分/件 5秒/件
各 PC のアドレス帳 個別管理・古い宛先が残る マスター複合機から自動配布
運用基盤 89 ローカル PC (単一障害点) 235 サーバー Docker (冗長化基盤の上)

得られた学び

  • 「いつでも見られる」は大きな価値 — トラブル時に5秒で確認できる体制は、業務の心理的な安心感を圧倒的に上げる
  • 89→235 Cutover の段階移行 — ローカル PC 運用から Linux サーバーへの移行は、責務分離 + 二重稼働期間 + 旧 URL リダイレクトの3段構えで安全に実施
  • Web UI と バックエンドの責務分離 — Prefect (バッチ) と Flask (UI) を分離することで、それぞれ独立して進化させられる

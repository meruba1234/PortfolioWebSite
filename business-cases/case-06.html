取引先C 月次請求金額内訳レポート

鮮魚系卸の取引先向けに、毎月の請求金額内訳を A4 縦 1 ページの Excel + PDF で自動生成。営業事務がブラックボックスから黒い画面で集計期間を入力するだけで完成。

業種
鮮魚卸 (取引先C)
対象部門
営業事務 (月次資料作成担当)
実装期間
約1.5週間
本番稼働
2026年6月
技術スタック
Python, SQL Server (readonly), openpyxl, win32com (PDF変換), Prefect 3.x, .bat 起動

背景・課題

取引先C (鮮魚卸) には毎月「ご請求金額内訳」資料を渡す必要があります。 対象は 4 得意先 (鮮魚 / 刺身 / 寿司 / 本社 = リベート) で、それぞれの:

  • 税抜金額
  • 消費税
  • 税込金額
  • 軽減税率(8%)対象額
  • 前年同期比 (税抜)

を、当年と前年を A4 縦 1 ページにまとめる必要があります。

本社 (リベート) は割戻料でマイナス計上、軽減税率の集計、前年同期比など、エクセル数式での集計だと毎月手で組み直す煩雑さがありました。

実装した解決策

1. 業務側からの起動

  • 営業事務が共有フォルダの .bat ファイルをダブルクリック
  • 黒い画面で集計期間 (開始月・終了月) を入力
  • 235 サーバーの Prefect API を叩いて Flow を起動
  • 完了するとレポートが共有フォルダに出力される

2. データ取得 (readonly DB)

  • 販売管理 DB (SQL Server) に readonly ユーザーで接続
  • 4 得意先の売上データを期間指定で集計
  • 税区分を判定して税抜・消費税・税込・軽減税率対象を分離
  • 前年同期分も同時取得して前年同期比を計算

3. レポート出力 (Excel + PDF)

  • openpyxl で Excel テンプレートに数値を流し込み
  • A4 縦 1 ページに収まるレイアウト調整
  • 担当 PC 側に Excel がある場合は、win32com で高品質 PDF を再生成 (2段構え)
  • Linux 側 (Prefect コンテナ内) は LibreOffice headless でフォールバック

4. アーキテクチャ

  • Flow 層 (本リポ) は Prefect 統合と subprocess 起動のみ
  • Script 層は別リポジトリ (batch-files) に分離
  • Script 層のロジックは他の月次レポートでも再利用可能

効果

項目BeforeAfter
レポート作成時間 約1時間/月 (手作業) 約2分 (bat 起動→完成)
集計ミスリスク 数式組み直しでミス発生しうる ゼロ (コードで一元管理)
PDF 品質 Excel→PDF 手動変換でレイアウト崩れあり 担当 PC で再生成して高品質維持
営業事務の操作 Excel 開いて集計・書式調整 bat ダブルクリック + 期間入力のみ

得られた学び

  • 業務側のスキルレベルに合わせた起動経路 — 「bat ダブルクリック + 黒い画面で入力」は技術的にはローテクだが、営業事務さんが迷わず使える最強の UI
  • PDF 品質の2段構え — Linux で生成した PDF はレイアウトが微妙なことがある。Windows 担当 PC で win32com 再生成する経路を持つことで、品質保証
  • readonly ユーザーでの DB 接続 — 集計系ジョブは絶対に書き込まないため、専用 readonly ユーザーを使う設計で誤更新事故をゼロに
